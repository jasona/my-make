# My Make

A prompt system for generating production-quality React UI components from a single freeform description — running entirely inside Claude Code, with no Figma dependency.

---

## What it does

Type `/make` with a plain-English description of what you want to build. My Make interprets your request, derives a structured design brief, and outputs a complete, runnable React + TypeScript + Tailwind component with a deliberate visual identity — not a generic template.

```
/make a loan pipeline dashboard card for a private mortgage lending SaaS
```

That's the entire invocation. Everything else — the brief, the aesthetic direction, realistic placeholder data, hover states, typography choices — is handled automatically.

---

## Commands

### `/make [description]`

The main entry point. Accepts any freeform description.

**What happens:**

1. Derives a full design brief from your description (component type, purpose, content, interactions, aesthetic direction, constraints)
2. Prints the interpreted brief in a collapsible block so you can see what it understood
3. Outputs a complete, runnable component
4. Offers to iterate

**If called with no arguments**, it asks one question: *"What do you want to make?"*

---

### `/iterate [changes]`

Surgical refinement of the current component. List what to change — nothing else will be touched.

**Example:**

```
/iterate make the stage badges solid fills and compress the card height
```

**If called with no arguments**, it surfaces the structured iteration template so you can be precise.

---

## Output guarantee

Every component generated by My Make:

- Is a **complete, runnable single file** — copy it into any Vite + React + Tailwind project and it works
- Uses **realistic placeholder data** — no "Item 1" or "Lorem ipsum"
- Has a **deliberate visual identity** — never a default shadcn/Tailwind template
- Renders with **zero props** (all defaults included)
- Is typed with **TypeScript** throughout

---

## Design system

My Make enforces a strict aesthetic contract. Things it will never produce:

- Inter, Roboto, Arial, or system-ui as the primary font
- Purple gradients on white backgrounds
- `rounded-xl shadow-md` cards as the primary layout unit
- Any layout that looks like a default Tailwind or shadcn template

Things it always considers:

- Distinctive font pairings (loaded from Google Fonts)
- Cohesive color palettes defined as CSS custom properties
- Framer Motion for meaningful animation (page-load stagger, hover elevation)
- Asymmetry, overlap, and grid-breaking elements when appropriate

---

## Aesthetic vocabulary

Use any of these terms in your `/make` prompt to set the visual direction:

| Direction | Reference |
|---|---|
| `editorial` | NYT, Stripe press — strong typography, generous whitespace, serif accents |
| `technical/dark` | Linear, Vercel, GitHub dark — high contrast, monospace, minimal decoration |
| `luxury-minimal` | Apple, Notion — extreme whitespace, single accent color, refined micro-typography |
| `high-density dashboard` | Bloomberg, Datadog — packed grid, small text, color as data encoding |
| `brutalist` | Are.na — raw structure, no softening |
| `warm-organic` | Craft, Loom — rounded but not childish, warm tones, friendly micro-copy |
| `glassy/blur` | macOS Sonoma, iOS — backdrop-filter blur, translucency, depth layers |
| `retro-futuristic` | Raycast, Arc — nostalgic palette with modern motion |

If you don't specify one, My Make picks the most appropriate direction for your context and tells you why.

## Prompt tips

**Specify what to avoid, not just what you want.**
Negative constraints are often more effective than positive descriptions:

> "Avoid centered hero layouts and gradient CTAs. I want a dense, information-first grid."

**Put hard constraints first.**
If the component has non-negotiable technical requirements, lead with them:

> "This must support virtualized lists of 50,000 rows and controlled selection via props. Now build a data table for..."

**Use real data shapes.**
Placeholder data quality directly affects layout decisions. The more realistic your field names and values, the better the output:

> `borrowerName: "Heritage Capital Partners LLC"` not `name: "John Doe"`

## Installation

**Prerequisites:** [Node.js](https://nodejs.org) and [Claude Code](https://claude.ai/code).

```bash
npm install
npm run dev
```

Open the URL Vite prints (usually `http://localhost:5173`). You'll see a placeholder screen. Now open Claude Code in this directory and start making:

```
/make a settings page for a developer tools SaaS — technical/dark aesthetic, desktop
```

Claude writes the component to `src/components/Current.tsx`. Vite hot-reloads instantly — no browser refresh needed.

### Optional — Enable stock photography

`/make` automatically fetches contextually relevant photos from Unsplash and embeds them in generated components. To enable it:

1. Go to [unsplash.com/developers](https://unsplash.com/developers), create a free app, and copy the **Access Key**
2. Create a `.env` file at the project root (already gitignored):
```
UNSPLASH_ACCESS_KEY=your_access_key_here
```
3. Done. The next `/make` will fetch real photos and embed them automatically.

The free tier allows 50 requests/hour — sufficient for a full design session. If no key is present, `/make` falls back to styled placeholder divs silently.

---

### How the live preview works

`src/App.tsx` renders a single file: `src/components/Current.tsx`. Every `/make` and `/iterate` command writes directly to that file. Vite watches it and pushes updates to the browser via HMR the moment the file is saved.

```
/make →  writes Current.tsx  →  browser updates
/iterate →  rewrites Current.tsx  →  browser updates
```

No copy-paste, no manual import changes, no refresh.

## Why not Figma Make?

Figma Make is Claude Sonnet routing through Figma's proxy, with Figma's token overhead on top. My Make calls the same model directly, with a tighter prompt system tuned for quality.

| | Figma Make | My Make |
|---|---|---|
| Token overhead | Figma proxy + model + retry loops | Model only |
| Output quality | Good, generic | Deliberately styled |
| Iteration control | Freeform | Structured (VISUAL / LAYOUT / BEHAVIOR / DATA) |
| Lock-in | Figma account required | None — plain files |
| Cost (20-screen SaaS) | Hits plan limits quickly | < $2 in API tokens |
